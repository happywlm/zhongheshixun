## 二、原始需求中高并发/架构要求 vs 作业简化方案

| 原始需求（生产级） | 规划文档（作业简化） | 说明 |
|-------------------|---------------------|------|
| CDN + 智能双路 + LVS/HA-Proxy 负载均衡 | 单体 Spring Boot | ✅ 作业合理简化 |
| 流媒体服务器分离（Nginx-RTMP + FFmpeg） | 本地存储 + 视频播放 | ✅ 作业合理简化 |
| 读写分离 + Sharding + 数据库集群 | 单机 MySQL | ✅ 作业合理简化 |
| MapReduce/BSP 分布式计算 | 无 | ✅ 作业合理简化 |
| IAAS/PAAS/SAAS 三层云服务架构 | 单体应用 + 设计文档逻辑分层描述 | ✅ 论文答辩口径见需求偏差说明.md |
| 移动端 APP (uniapp/Flutter) | 微信小程序原生 | ✅ 作业调整（更轻量） |
| 离线学习：断点续传 + 进度回传 + ZIP 加密 | 字段层（course.offline_flag + zip_url）实现；**manifest 打包脚本 + 进度批量回传 /api/study/batch 未实现** | ⚠️ 功能简化较多，有设计预留 |
| 安全：传输加密 + 脱敏 + 权限控制 | JWT 基础认证 + BCrypt | ⚠️ 安全要求大幅简化 |
