## 三、开发过程中的问题与教训

### 3.1 技术问题

| 问题 | 影响 | 解决方案 | 教训 |
|------|------|---------|------|
| 8081 端口僵尸死锁 | 无法启动 training-api | 迁移至 9899 | 端口冲突时不要硬杀，直接换端口 |
| 5174 端口双进程冲突 | 前端启动混乱 | 管理后台固定 5176 | 多前端项目必须明确端口分配 |
| train-plan/page 500 | 培训计划不可用 | 补 plan 模块权限 | 权限字典必须与 Controller 注解完全一致 |
| MyBatis-Plus 分页 count 缺失 | total=0 | Controller 兜底 | 自定义分页查询需手动处理 total |
| Spring Security 鉴权失败无日志 | 排查困难 | 改 filter 为 log.error + sendError | 鉴权异常必须打完整堆栈 |

### 3.2 流程问题

| 问题 | 影响 | 解决方案 | 教训 |
|------|------|---------|------|
| 子代理零文件崩溃 | 浪费时间 | 主线程接管关键代码 | 关键代码不要依赖子代理 |
| 文档与代码不同步 | 答辩时端口写错 | 每次代码变更同步文档 | 文档即代码，同步更新 |
| 需求理解偏差 | 需求偏差说明 v1.0 仅 85% | 修订为 v2.0 100%覆盖 | 需求必须逐条对照代码验证 |

### 3.3 经验总结

1. **多代理协作**：tech-director 统稿 + 3 个 Lead 分工，效率远高于单人开发
2. **阶段门控**：设计 → 编码 → 集成 → 测试，每阶段评审，避免返工
3. **文档驱动**：先写设计文档，再编码，减少需求理解偏差
4. **真实数据**：文档中所有端口、表名、字段名必须与代码一致，不能编造
5. **问题记录**：开发过程问题记录表是宝贵的答辩素材
