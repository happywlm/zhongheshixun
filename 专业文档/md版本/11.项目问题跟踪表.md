## 问题明细

### BUG-001｜小程序"我的"永远进登录页
- **模块：** 小程序个人中心
- **现象：** 用户点击 tabBar 的"我的"进入的是登录页
- **根因：** `app.json` tabBar 的"我的"指向 `pages/login/login`，无独立 personal 页面
- **修复：** 新建 `pages/profile/profile` 4 文件；tabBar 改为指向新页面
- **验证：** ✅ 点击"我的" → 展示 Profile 页（未登录则跳登录）

### BUG-002｜首页入口链接全错
- **模块：** 小程序首页
- **现象：** 点击"考试""计划""我的"都跳登录页
- **根因：** `pages/index/index.js` 中 entries 表 3 项指向登录页
- **修复：** entries 全部改为真实页面，`onGoTo` 区分 tabBar 页 → switchTab / 普通页 → navigateTo
- **验证：** ✅ 各入口均正确导航

### BUG-003｜启动强制跳登录
- **模块：** 小程序 app.js
- **现象：** 启动立即跳到登录页，tabBar 看不到
- **根因：** `app.js` 启动时强制跳转登录页
- **修复：** 移除强制跳转，改为 tabBar 正常展示
- **验证：** ✅ 启动后展示 tabBar 首页

### BUG-004｜train-plan/page 返回 500
- **模块：** 培训计划
- **现象：** GET /admin/train-plan/page 返回 500 "系统繁忙，请稍后重试"，无 SQL 日志、无 Exception 日志
- **根因：** `sys_permission` 表缺少 `plan:read`/`plan:write` 权限记录，`@PreAuthorize("hasAuthority('plan:read')")` 鉴权失败，异常被 filter 旧 catch 吞掉
- **修复：** 补 `plan:read`/`plan:write` 权限 + role_permission 关联；改 filter 为 log.error + sendError(401)
- **验证：** ✅ 培训计划分页正常，数据可见

### BUG-005｜课程封面为空
- **模块：** 培训计划
- **现象：** web-student 课程卡显示空占位
- **根因：** course 表 coverUrl 字段为空
- **修复：** UPDATE course SET coverUrl='https://picsum.photos/seed/courseN/400/225'
- **验证：** ✅ 课程卡显示封面图

### BUG-006｜端口冲突 5174 双进程
- **模块：** 系统
- **现象：** netstat 显示 5174 端口上有 2 个 node 进程
- **根因：** web-student 和管理后台前端都配置 port=5174
- **修复：** 管理后台前端改为 5176
- **验证：** ✅ 5174（web-student）+ 5176（管理后台）稳定运行

### BUG-007｜8081 端口僵尸死锁
- **模块：** 系统
- **现象：** 8081 端口被僵尸进程内核级死锁，所有 kill 工具均失败
- **根因：** Windows 内核级进程残留
- **修复：** 放弃 8081，training-api 迁移至 9899
- **验证：** ✅ 9899 端口正常

### BUG-008｜小程序 mock 数据未切换真接口
- **模块：** 小程序
- **现象：** 小程序显示 mock 数据，不调用后端
- **根因：** api/course.js 和 api/exam.js 里 mock 块未注释
- **修复：** 取消注释 request(...) 调用，注释 mock 块
- **验证：** ✅ 小程序调用真实后端接口

### BUG-009｜小程序考试提交无响应
