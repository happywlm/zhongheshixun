## 一、引言

### 1.1 编写目的
本文档定义《四川省基层卫生人员网络培训平台》的配置管理流程，规定配置项识别、基线划分、变更控制、状态记录、审核机制，确保代码与文档可追溯、可复现。

### 1.2 项目背景
本平台面向四川省 67 个民族县 3000 名基层卫生技术人员，采用 Spring Boot + Vue 3 + 微信小程序 技术栈，开发周期 6 周（2026-06-30 ~ 2026-08-10），采用 Claude Code 多代理协作开发模式。

## 五、变更控制

### 5.1 变更分类

| 分类 | 描述 | 审批 |
|------|------|------|
| 重大变更 | 架构调整、接口契约变更、数据库 Schema 变更 | CCB 评审 |
| 一般变更 | Bug 修复、页面样式调整 | tech-director 审批 |
| 微小变更 | Typo、注释 | 开发组自审 |

### 5.2 CR 流程

```
1. 提交 CR（Change Request）
   ↓
2. tech-director 评审
   ├─ 重大变更 → 召开 CCB 会议，记录决策
   └─ 一般变更 → 直接审批
   ↓
3. 实施变更
   ↓
4. 同步更新受影响配置项（文档/代码/脚本）
   ↓
5. 验证 + 提交变更记录
```

| 类型 | 频率 | 保留 |

### 6.2 备份命令

```bash

git add . && git commit -m "feat: 描述" && git push origin dev

# MySQL 全量备份

```

### 6.3 恢复流程

