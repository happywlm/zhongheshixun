## 一、引言

### 1.1 编写目的
本文档定义《四川省基层卫生人员网络培训平台》的完整功能需求，作为开发、测试、验收的基准。

### 1.2 项目背景
本项目以提高四川省 67 个民族县基层卫生人员业务能力为目标，面向乡镇卫生院和社区卫生服务中心的七类卫生技术人员（临床医学、公共卫生、护理、医技B超/心电/检验/影像），共 3000 人，每人每年需完成 200 学时。建设集"在线学习、在线考试、离线学习、智能问答、实时统计"于一体的网络培训平台。

## 二、总体描述

### 2.1 系统边界

**系统内：**
- 学员管理（小程序用户 + 系统管理员维护的学员账号）
- 课程/章节/资料
- 考试/试卷/试题
- 咨询工单/知识库
- 培训计划
- 统计分析

**系统外：**
- 微信小程序开放平台（登录授权）
- 第三方对象存储（视频/资料文件）

### 2.2 用户角色

| 角色 | 描述 | 主要操作 |
|------|------|---------|
| 系统管理员 | 卫健委管理人员 | 课程/试题/考试/用户/培训计划/统计 |
| 培训讲师 | 医院/医学院教师 | 课程管理、试题管理、答疑 |
| 学员 | 基层卫生技术人员 | 学习、考试、咨询、查看进度 |

### 2.3 运行环境

| 维度 | 配置 |
|------|------|
| 后端 | JDK 17.0.9 + Spring Boot 2.7.18 |
| 数据库 | MySQL 8.0.41，库名 `training`，utf8mb4 |
| 前端 | Node.js 18+ + Vue 3 + Vite |
| 小程序 | 微信开发者工具 1.06+ |
| OS | Windows 11 22631（开发）/ Linux（生产） |

### 2.4 技术约束
- 单体 Spring Boot 应用（非微服务）
- 三端（管理后台 + Web 学员端 + 小程序）共享同一数据库
- JWT 统一身份认证，RBAC 细粒度鉴权
- 响应时间 < 2 秒（演示机环境）

## 四、非功能需求

### 4.1 性能需求
- 支持 3000 用户日常并发学习
- 支持 1000 并发考试
- 单请求响应时间 < 2 秒（演示机环境）
- 数据库连接池：HikariCP，最大 20 连接

### 4.2 安全需求
- 密码 BCrypt 加密存储
- JWT 过期时间 7 天
- 所有 `/admin/**` 路径要求 ROLE_ADMIN
- 方法级 `@PreAuthorize` 细粒度鉴权
- SQL 注入防护：MyBatis `#{}` 参数绑定
- XSS 防护：UGC 字段转义

### 4.3 兼容性需求
- 浏览器：Chrome 120+ / Edge 120+ / Firefox 115+
- 移动端：iOS 14+ / Android 8.0+
- 微信：iOS 8.0.30+ / Android 8.0.30+

### 4.4 数据需求
- 数据库：MySQL 8.0，utf8mb4
- 逻辑删除：deleted 字段（0=正常，1=已删）
- 自动填充：create_time / update_time
- 备份：每日全量备份（生产环境）

## 五、数据流

### 5.1 核心业务流程

```
学员报名课程：
  小程序 → POST /api/study/enroll → course_enroll 表 → 200

学员学习章节：
  Web/小程序 → POST /api/study/progress → study_record 表 → 进度聚合

学员参加考试：
  小程序 → POST /api/exam/start/{id} → exam_record 表
        → POST /api/exam/submit → 自动阅卷 → exam_answer 表 → 当场返分

管理员发布课程：
  管理后台 → POST /admin/course → course 表 → 学员端可见

咨询 SLA 超时：
  学员提问 → knowledge_base 匹配 → 转人工 → 超过 1 分钟 → sla_exceeded=1 → 后台红色高亮
```
