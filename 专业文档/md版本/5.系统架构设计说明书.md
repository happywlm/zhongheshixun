## 一、引言

### 1.1 编写目的
本文档定义《四川省基层卫生人员网络培训平台》的软件架构，包括技术选型、分层结构、模块交互、数据流、安全性、可维护性设计，作为编码与测试的顶层约束。

### 1.2 项目背景
本平台面向四川省 67 个民族县 3000 名基层卫生技术人员，提供 MOOC 在线学习、在线考试、离线学习、智能问答、实时统计等服务。

### 1.3 架构总览

本平台采用 **单体 Spring Boot 应用 + 双独立前端 + 微信小程序** 架构：

```
学员端                             管理端
┌──────────────────────┐    ┌──────────────────────────┐
│ 微信小程序（学员移动） │    │ Vue3 + Element Plus（PC） │
│  端口 -               │    │  端口 5176（管理后台）    │
│  端口 5174（Web 端）  │    │                          │
└────────┬─────────────┘    └────────────┬─────────────┘
         │  HTTPS API                    │  HTTPS API
         ▼                               ▼
┌────────────────────────────────────────────────────────┐
│  training-api 应用（Spring Boot 9899）                 │
│  - RESTful API + JWT 认证（无 RBAC）                  │
│  - 7 个 Controller（course/exam/study/consult/...）   │
└────────────────────────────┬───────────────────────────┘
                             │  JDBC (HikariCP)
                             ▼
                   ┌──────────────────────┐
                   │  MySQL 8.0（单机）    │
                   │  - training 库       │
                   │  - 20 张表           │
                   │  - utf8mb4           │
                   └──────────────────────┘
                             ▲
                             │  JDBC (HikariCP)
┌────────────────────────────┴───────────────────────────┐
│  training-admin 应用（Spring Boot 9898）               │
│  - RESTful API + JWT + RBAC                           │
│  - Spring Security + @PreAuthorize                     │
│  - 11 个 Controller + 7 个统计接口                    │
└────────────────────────────────────────────────────────┘
```

> **架构决策**：原始需求提出 IAAS/PAAS/SAAS 三层云服务、LVS/HA-Proxy 负载均衡、流媒体服务器分离。作为本科毕业设计，本平台简化为**单体 MySQL + 单体 Redis（可选）+ 单体 Spring Boot**，核心业务流程 100% 实现；简化点已在《需求偏差说明 v2.0》中逐项说明，可接受。

## 五、核心业务流

### 5.1 鉴权流程

```
请求
  │
  ▼
JwtAuthenticationFilter（OncePerRequestFilter）
  │ 1. 白名单放行（/admin/login、/api/wx/login、/error）
  │ 2. 取 Authorization Header
  │ 3. jwtUtils.validate(token) → 校验签名+过期
  │ 4. claims.get("username") → 加载 UserDetails
  │ 5. 构建 UsernamePasswordAuthenticationToken 写入 SecurityContext
  │ 任何异常 → log.error + response.sendError(401) + return
  ▼
Spring Security Filter Chain
  │ FilterSecurityInterceptor：URL 粗粒度 /admin/** requires ROLE_ADMIN
  ▼
Method Security Interceptor
  │ @PreAuthorize("hasAuthority('course:read')") 方法细粒度
  ▼
Controller
  │ 业务逻辑
  ▼
Result<T> 统一响应 JSON
```

### 5.2 自动阅卷流程

```
POST /api/exam/submit
  │
  ▼
ExamBizServiceImpl.autoGrade(List<Answer> userAnswers, List<Question> dbAnswers)
  │
  ├─ 遍历 userAnswers：
  │   ├─ type=1/3（单选/判断）：equals 比较
  │   ├─ type=2（多选）：转 Set 比较（无序）
  │   ├─ type=4（填空）：split("\\|") 命中其一即对
  │   └─ type=5（问答）：跳过，is_correct=null
  │
  ├─ 统计 correct / wrong
  ├─ score = (correct / total) × 100
  ├─ passed = score >= 60
  │
  ├─ INSERT exam_record（exam_id, student_id, submit_time, score, passed）
  ├─ INSERT exam_answer 明细（record_id, question_id, user_answer, is_correct）
  │
  └─ 返回 ExamResultVO{ score, passed, correctCount, wrongCount, details }
```

### 5.3 咨询 SLA 告警流程

```
学员提问
  │
  ▼
knowledge_base FULLTEXT 匹配（ngram 中文分词）
  │
  ├─ 命中 → is_auto=1，直接返回答案
  └─ 未命中 → is_auto=2，insert consult_record（status=待回复）
       │
       ▼
     管理员回复（POST /admin/consult/reply）
       │
       计算耗时 = reply_time - create_time
       │
       ├─ 耗时 < 60 秒 → sla_exceeded=0
       └─ 耗时 >= 60 秒 → sla_exceeded=1
           │
           ▼
         GET /admin/consult/sla-alert 返回超时工单列表
         后台 consult 页红色高亮显示
```

## 八、安全设计

### 8.1 认证
- JWT 过期时间 7 天（training.jwt.expire=604800000）
- 密钥：`training-platform-jwt-secret-key-2026-07-su-sheng-ji-ceng-wei-sheng`
- 三端密钥完全一致，token 可跨端使用

### 8.2 授权
- URL 粗粒度：SecurityConfig `antMatchers("/admin/**").hasRole("ADMIN")`
- 方法细粒度：`@PreAuthorize("hasAuthority('course:read')")`
- 401 走 CustomAuthenticationEntryPoint（返回 JSON `{"code":401}`）
- 403 走 CustomAccessDeniedHandler（返回 JSON `{"code":403,"message":"无权限访问"}`）

### 8.3 数据安全
- 密码 BCrypt（成本因子 10）
- SQL 注入防护：全部 `#{}`
- 逻辑删除：deleted 字段，不物理删数据
- 敏感字段过滤：password、deleted 不进 DTO
