## 一、引言

### 1.1 编写目的
本文档描述《四川省基层卫生人员网络培训平台》的数据库逻辑设计与物理设计，作为数据库实施、前端开发、后端开发、测试人员的底层依据。

### 1.2 项目背景
本项目面向四川省 67 个民族县 3000 名基层卫生技术人员，提供 MOOC 在线学习、在线考试、离线学习、智能问答、实时咨询等服务。系统基于 Spring Boot + Vue 3 + 微信小程序 构建单体应用，数据库选用 MySQL 8.0。

### 1.3 术语定义

| 术语 | 含义 |
|------|------|
| MOOC | Massive Open Online Course，大规模开放在线课程 |
| JWT | JSON Web Token，无状态身份认证令牌 |
| SLA | Service Level Agreement，服务等级协议 |
| MP | MyBatis-Plus，增强版 MyBatis ORM 框架 |
| BCrypt | 单向哈希加密算法，用于密码存储 |
| 逻辑删除 | 用 deleted 字段标记删除状态，数据不物理移除，保证可追溯性 |
| 必修/选修 | 课程类型，1=公开课（选学），2=必修课（必学） |
| ngram | MySQL 全文索引解析器，支持中文分词 |

## 二、数据库总体设计

### 2.1 整体架构

本平台数据库采用 **20 张表**，按业务域划分为 6 个模块：

| 模块 | 表数 | 表名 |
|------|------|------|
| 用户与权限 | 4 | sys_user、sys_role、sys_permission、sys_role_permission |
| 课程与资源 | 5 | teacher、course、course_chapter、knowledge_point、resource_file |
| 学习 | 2 | course_enroll、study_record |
| 考试 | 4 | question、exam、exam_paper、exam_record、exam_answer |
| 咨询 | 2 | consult_record、knowledge_base |
| 培训计划 | 2 | train_plan、plan_course |
| 统计视图 | 1 | stats（统计聚合视图，代码中体现为 StatsMapper 查询） |

### 2.2 命名规范
- 表名：全小写 + 下划线（`sys_user`、`course_chapter`）
- 字段名：全小写 + 下划线（`user_id`、`course_type`）
- 主键：`id` BIGINT AUTO_INCREMENT
- 索引：`idx_字段名`、`uk_字段名`（唯一索引）
- 所有表统一字段：id、create_time、update_time、deleted

### 2.3 统一字段约束

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| deleted | TINYINT | NOT NULL DEFAULT 0 | 逻辑删除：0=正常，1=已删 |

## 八、安全设计

1. **密码安全**：BCrypt 加密，成本因子 10
2. **逻辑删除**：所有表 deleted 字段，数据不物理移除
3. **敏感字段**：password、deleted 不返回给前端（DTO 过滤）
4. **SQL 注入防护**：MyBatis 全部 `#{}` 参数绑定，禁用 `${}`
5. **权限隔离**：RBAC 三表 + @PreAuthorize 方法级鉴权
