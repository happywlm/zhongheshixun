# 学生丁 — 咨询问答 + 小程序（交互亮点 + 移动端）

> **综合实训角色**：学生丁  
> **业务域**：咨询问答 + 小程序  
> **核心叙事**：「学员提问题 → 机器先答 → 答不了转人工 → 人工 <1min 响应 → 小程序怎么用」

---

## 一、业务定位

你是整组**对外展示效果最好**的角色。你的模块是：

1. **交互亮点** — ⭐ 关键词匹配 + SLA 超时告警（docx 3.5 强制）
2. **移动端闭环** — ⭐ 小程序全 15 页（对外演示震撼）
3. **响应式保障** — ⭐ 应答时间 < 24 小时（SLA 告警，演示用 slaHours=24）

回答三个问题：

1. **学员在咨询问什么问题？** — 在线咨询（小程序 + web-student）
2. **系统怎么回答？** — ⭐ 关键词提取 + 知识库 LIKE 命中
3. **答不了怎么办？** — ⭐ 转人工 + 超时标红（SLA 保障）

前置依赖：
- **学生甲** 提供登录 token 和 userId（咨询需要用户身份）
- **学生乙** 的课程、**学生丙** 的试题可能被咨询引用

## 三、数据库表详解

### 3.1 knowledge_base（知识库表）⭐ 智能问答核心

| 字段 | 类型 | 说明 | 重要性 |
|------|------|------|--------|
| id | BIGINT PK | 知识条目 ID | ⭐ |
| question | VARCHAR(500) | ⭐ 问题（如"高血压怎么用药"） | ⭐⭐⭐ 匹配用 |
| answer | TEXT | ⭐ 标准答案 | ⭐⭐⭐ |
| keywords | VARCHAR(200) | ⭐ 关键词（逗号分隔，如"高血压,降压药"） | ⭐⭐⭐ 匹配用 |
| category | VARCHAR(50) | 分类 | ⭐⭐ |
| create_time | DATETIME | 创建时间 | ⭐ |
| update_time | DATETIME | 更新时间 | ⭐ |
| deleted | TINYINT | 逻辑删除 | ⭐⭐ |

> ⚠️ 实际无 `title`/`click_count`/`status` 字段。`question` 字段即问题文本
> （非 FAQ 标题），无点击热度统计（按 keywords LIKE 命中即返回）。
> 启停控制通过 `deleted` 软删除实现。

### 3.2 consult_record（咨询记录表）⭐ SLA 告警核心

| 字段 | 类型 | 说明 | 重要性 |
|------|------|------|--------|
| id | BIGINT PK | 记录 ID | ⭐ |
| student_id | BIGINT FK → sys_user.id | 学员 ID | ⭐⭐⭐ |
| question | VARCHAR(500) | 学员提问 | ⭐⭐⭐ |
| answer | TEXT | 回答（机器/人工） | ⭐⭐⭐ |
| is_auto | TINYINT | ⭐ 1 智能回答 / 2 人工回答 / 0 待处理 | ⭐⭐⭐ |
| create_time | DATETIME | ⭐ 创建时间（SLA 计时起点） | ⭐⭐⭐ |
| reply_time | DATETIME | ⭐ 回复时间（SLA 计时终点） | ⭐⭐⭐ |
| deleted | TINYINT | 逻辑删除 | ⭐⭐ |

> ⚠️ 实际无 `status`/`transfer_time`/`sla_seconds`/`sla_exceeded`/`admin_id`/`update_time` 字段。
> - 状态用 `is_auto` 三态表达：0 待处理（转人工未回复）/ 1 智能回答 / 2 人工回答；
> - SLA 通过 `create_time` 与 `reply_time` 的差值动态计算（见 4.2 节）；
> - 处理人通过 admin token 间接关联（人工回复时由 ConsultService 内部记录）。

### 3.3 表关系图

```
knowledge_base ──── 运行时 keywords LIKE 命中 ──── consult_record
（无外键，仅运行时匹配）                              │
                                                    │ is_auto=0 待处理

                                                    │ AND now - create_time > slaHours
                                                    ▼
                                              ⭐ /admin/consult/sla-alert 接口
                                                    │
                                                    ▼

sys_user (1) ──────────────────── (N) consult_record.student_id

consult_keyword ──── 学员提问命中 to_human 关键词 ──── 直接转人工（跳过 AI）
（6 条预置：转人工/找老师/人工客服/真人/人工/客服）
```

**⭐ 关键代码**：

```java
// ConsultServiceImpl.ask(userId, question)
public AskResult ask(Long userId, String question) {
    // 1. 命中 consult_keyword 路由（to_human 关键词直接转人工）
    if (consultKeywordMapper.matchHumanKeyword(question) > 0) {
        ConsultRecord record = new ConsultRecord();
        record.setStudentId(userId);
        record.setQuestion(question);
        record.setAnswer("正在为您转接人工客服...");
        record.setIsAuto(0);  // 待处理
        consultRecordMapper.insert(record);
        return new AskResult(record.getId(), false, false, "正在为您转接人工客服...");
    }

    // 2. 知识库关键词 LIKE 命中
    List<String> keywords = extractKeywords(question);
    // "高血压怎么用药" → ["高血压", "用药"]
    List<KnowledgeBase> hits = knowledgeBaseMapper.searchByKeywords(keywords);

    if (!hits.isEmpty()) {
        // 命中：返回预设答案，is_auto=1
        KnowledgeBase bestMatch = hits.get(0);
        ConsultRecord record = new ConsultRecord();
        record.setStudentId(userId);
        record.setQuestion(question);
        record.setAnswer(bestMatch.getAnswer());
        record.setIsAuto(1);  // 智能回答
        consultRecordMapper.insert(record);
        return new AskResult(record.getId(), true, true, bestMatch.getAnswer());
    }

    // 3. 无匹配 → 转人工（is_auto=0 待处理，等待管理员回复）
    ConsultRecord record = new ConsultRecord();
    record.setStudentId(userId);
    record.setQuestion(question);
    record.setAnswer("正在为您转接人工客服...");
    record.setIsAuto(0);  // 待处理
    consultRecordMapper.insert(record);
    return new AskResult(record.getId(), false, false, "正在为您转接人工客服...");
}

// SQL 关键词匹配（无 status/click_count 字段，按 deleted 软删除过滤）
// <select id="searchByKeywords">
//   SELECT * FROM knowledge_base
//   WHERE deleted = 0

//     OR keywords LIKE CONCAT('%', #{kw}, '%')
//     OR question LIKE CONCAT('%', #{kw}, '%')
```

### 4.2 ⭐ SLA 超时告警机制（ConsultController，9898）⭐ 学习重点

**SLA 告警落地方案**：

```
1. consult_record 两字段记录时间：
   - create_time：咨询创建时间（SLA 计时起点）
   - reply_time：人工回复时间（SLA 计时终点，未回复时为 NULL）

2. 管理员后台调用 /admin/consult/sla-alert?slaHours=24 接口：
   SELECT * FROM consult_record
   WHERE is_auto = 0           -- 待处理（转人工未回复）
     AND reply_time IS NULL    -- 未回复
     AND TIMESTAMPDIFF(HOUR, create_time, NOW()) > slaHours;
   返回的记录即"超时告警列表"。

3. 前端 consult/list.vue 调 /sla-alert 拿超时列表 → 红色标红显示
```

**接口签名**（ConsultController.slaAlert）：

```java
@PreAuthorize("hasAuthority('consult:read')")
@GetMapping("/sla-alert")
public Result<List<ConsultRecord>> slaAlert(
        @RequestParam(defaultValue = "24") int slaHours) {
    List<ConsultRecord> list = consultService.getOverdueConsults(slaHours);
    return Result.success(list);
}
```

**前端代码**（consult/list.vue，简化示意）：

```vue
<template>
  <el-table :data="consultList">
    <el-table-column prop="question" label="问题" />
    <el-table-column prop="createTime" label="提问时间" />
    <el-table-column prop="replyTime" label="回复时间" />
    <el-table-column prop="isAuto" label="状态">
      <template #default="{ row }">
        <el-tag :type="row.isAuto === 0 && !row.replyTime ? 'danger' : 'success'">
          {{ row.isAuto === 0 && !row.replyTime ? '超时未回复' : '已回复' }}
        </el-tag>
      </template>
    </el-table-column>
  </el-table>
</template>
```

### 5.3 小程序关键技术点

**智能问答聊天 UI**（现代大模型风格）：

```xml
<!-- consult/index.wxml -->
<view class="chat-container">
  <scroll-view scroll-y class="chat-list">
    <block wx:for="{{messages}}" wx:key="id">
      <!-- 学员消息 -->
      <view class="msg user" wx:if="{{item.role == 'user'}}">
        <view class="bubble">{{item.content}}</view>
      </view>
      <!-- 客服/系统消息 -->
      <view class="msg bot" wx:else>
        <view class="bubble">{{item.content}}</view>
      </view>
    </block>
    <!-- 打字机效果 -->
    <view class="msg bot" wx:if="{{isTyping}}">
      <view class="bubble typing">····</view>
    </view>
  </scroll-view>
  
  <view class="input-bar">

    <button bindtap="onSend" disabled="{{isTyping}}">发送</button>
  </view>
```

```javascript
// consult/index.js
Page({
  data: {
    messages: [],
    input: '',
    isTyping: false
  },
  
  onSend() {
    const question = this.data.input;
    if (!question.trim()) return;
    
    // 添加学员消息到列表
    this.addMessage('user', question);
    
    // 显示打字机效果
    this.setData({ isTyping: true, input: '' });
    
    // 模拟打字机 1.5 秒
    setTimeout(() => {
      // 调 API
      wx.request({
        url: 'http://localhost:9899/api/consult/ask',
        method: 'POST',
        header: { Authorization: 'Bearer ' + wx.getStorageSync('token') },
        data: { question },
        success: (res) => {
          const answer = res.data.data.answer;
          this.addMessage('system', answer);
          this.setData({ isTyping: false });
        }
      });
    }, 1500);
  },
  
  addMessage(role, content) {
    const messages = this.data.messages;
    messages.push({
      id: Date.now(),
      role,
      content,
      time: this.formatTime()
    });
    this.setData({ messages });
    // 自动滚到底部
```

## 九、常见 Bug 与解决

| 现象 | 原因 | 解决 |
|------|------|------|
| 关键词没命中 | keywords 字段没写或分隔符错 | 检查是英文逗号 |
| 命中了但返回错答案 | searchByKeywords LIMIT 1 取错 | 看 SQL 排序 |
| 转人工没写 DB | ask() 漏 insert 待处理 | 看 ConsultServiceImpl |
| 标红没生效 | /sla-alert 没调或参数错 | 看 consult/list.vue 接口调用 |
| 回复后 is_auto 没变 | 后端漏 update is_auto=2 | 看 reply() 流程 |
| 小程序 401 | token 没带或过期 | 看 wx.getStorageSync('token') |
| 小程序网络失败 | baseURL 写错 | 看 app.js globalData.apiBase |
| 小程序视频播不了 | 视频 URL 是 localhost 非 HTTPS | 开发工具勾选"不校验合法域名" |
| PDF 下载失败 | file_url 路径错 | 看 resource_file.file_url |
| 打字机没效果 | isTyping 状态没 toggle | 看 setData 顺序 |
