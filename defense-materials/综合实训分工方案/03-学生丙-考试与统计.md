# 学生丙 — 考试与统计（算法核心 + 综合实训亮点）

> **综合实训角色**：学生丙  
> **业务域**：考试 + 统计  
> **核心叙事**：「试题怎么入库 → 试卷怎么组 → 学员怎么考 → 机器怎么判 → 成绩怎么分析」

## 一、业务定位

你是整组**技术深度最高**的角色。你的模块集中了**两大核心算法 + 多角度统计**，是综合实训最被追问的部分：

1. **试题怎么入库？** — 5 题型 + 3 难度 + 知识点归属
2. **试卷怎么组？** — ⭐ **自动组卷**（30:50:20 难度比）
3. **学员怎么考？** — 双保险计时（前端倒计时 + 后端 receiveTime）
4. **机器怎么判？** — ⭐ **自动阅卷**（4 题型精确匹配）
5. **成绩怎么分析？** — ⭐ 7 统计接口 + ECharts 三图 + 4 维度统计表格

前置依赖：
- **学生甲** 提供登录 token 和 userId（考试需要用户身份）
- **学生乙** 的课程提供考试关联（exam.course_id → course.id）

### 4.4 ⭐ 自动组卷算法（ExamServiceImpl.generatePaper）

**代码位置**：`training-admin/src/main/java/com/training/admin/service/impl/ExamServiceImpl.java`

**⭐ 核心算法（手抄必会）**：

```java
/**
 * 自动组卷：按 30:50:20 难度比随机抽题 + shuffle
 * @param examId 考试 ID
 * @param count  题目总数
 * @return 打乱顺序的题目列表
 */
public List<Question> generatePaper(Long examId, int count) {
    // 1. 按比例计算题量
    int easy   = count * 30 / 100;  // 30% 简单
    int medium = count * 50 / 100;  // 50% 中等
    int hard   = count - easy - medium;  // 剩余给困难（避免舍入误差）
    
    // 2. 各难度随机抽题
    List<Question> questions = new ArrayList<>();
    questions.addAll(questionMapper.randomByDifficulty(1, easy));
    questions.addAll(questionMapper.randomByDifficulty(2, medium));
    questions.addAll(questionMapper.randomByDifficulty(3, hard));
    
    // 3. shuffle 打乱顺序
    Collections.shuffle(questions);
    
    // 4. 写 exam_paper 表
    ExamPaper paper = new ExamPaper();
    paper.setExamId(examId);
    paper.setQuestions(JSON.toJSONString(questions));  // JSON 数组
    paperMapper.insert(paper);
    
    return questions;
}
```

**SQL 实现（Mapper XML，教学示例）**：

> ⚠️ 实际实现：ExamServiceImpl 通过 `question_ids` 字段预组卷，或运行时按 courseId + difficulty 抽题。下方 Mapper XML 为教学简化版。

### 4.3 学员侧 — 开始考试（ExamApiController，9899）⭐ 双保险计时

**请求**：

```http
POST /api/exam/start/{id}
Authorization: Bearer {student_token}
```

**关键代码**：

```java
@PostMapping("/start/{id}")
public Result start(@PathVariable Long id, HttpServletRequest request) {
    Long studentId = (Long) request.getAttribute("userId");
    // 1. 查考试
    Exam exam = examMapper.selectById(id);
    
    // 2. 查最近答卷（status=0 进行中且未超时则继续考）
    ExamRecord lastRecord = recordMapper.selectOne(
        new QueryWrapper<ExamRecord>()
            .eq("exam_id", id)
            .eq("student_id", studentId)
            .eq("status", 0)
            .orderByDesc("create_time")
            .last("LIMIT 1")
    );
    if (lastRecord != null && !isTimeout(lastRecord, exam.getDuration())) {
        // 继续考：复用进行中记录
        return Result.success(lastRecord);
    }
    // 超时的 status=0 记录标记为 status=2（已批阅/超时）
    if (lastRecord != null) {
        lastRecord.setStatus(2);
        recordMapper.updateById(lastRecord);
    }
    
    // 3. 组卷（调 generatePaper 或读取预组卷 question_ids）
    List<Question> questions = examService.generatePaper(id, exam.getQuestionCount());
    
    // 4. 建考试记录（⭐后端记录 start_time）
    ExamRecord record = new ExamRecord();
    record.setStudentId(studentId);
    record.setExamId(id);
    record.setStatus(0);  // 进行中
    record.setStartTime(new Date());  // ⭐ 后端计时起点
    recordMapper.insert(record);
    
    return Result.success(record);
}
```

**⭐ 双保险计时机制（必画）**：

```
前端计时                    后端计时
   │                          │
   │ 从 response.startTime    │ start_time 存 DB
   │ 开始倒计时               │
   │                          │
   │ setInterval 每秒 -1      │
   │                          │
   │ 倒计时到 0 → 自动交卷    │ 收到 submit 时算 duration
   │                          │ = submit_time - start_time
   │ 提交                      │ 如果 duration > total_time
   │ POST /api/submit          │ → 拒绝 or 按总分判
   │                          │
   └──────── 交卷 ────────────┘
   
结论：即使学员改本地时间、断网，后端都以 receiveTime 为准
```

### 4.6 ⭐ 自动阅卷算法（ExamServiceImpl.autoGrade）⭐ 核心

**代码位置**：`training-api/src/main/java/com/training/api/service/impl/ExamServiceImpl.java`

**⭐ 核心算法（手抄必会）**：

```java
/**
 * 自动阅卷：4 客观题精确匹配，问答题跳过
 * @param paperId     试卷 ID
 * @param userAnswers 学员答案列表
 */
public ExamResult autoGrade(Long paperId, List<AnswerDTO> userAnswers) {
    // 1. 查试卷 + 题目
    ExamPaper paper = paperMapper.selectById(paperId);
    List<Question> dbQuestions = parseQuestions(paper.getQuestions());
    Map<Long, Question> questionMap = dbQuestions.stream()
        .collect(Collectors.toMap(Question::getId, q -> q));
    
    int correct = 0, wrong = 0, skipped = 0;
    BigDecimal totalScore = BigDecimal.ZERO;
    List<ExamAnswer> records = new ArrayList<>();
    
    // 2. 逐题判分
    for (AnswerDTO userAns : userAnswers) {
        Question dbQ = questionMap.get(userAns.getQuestionId());
        if (dbQ == null) { wrong++; continue; }
        
        // 建答题记录
        ExamAnswer record = new ExamAnswer();
        record.setRecordId(paperId);
        record.setQuestionId(dbQ.getId());
        record.setUserAnswer(userAns.getContent());
        record.setCorrectAnswer(dbQ.getAnswer());
        
        boolean isCorrect = false;
        
        switch (dbQ.getType()) {
            case 1: // 单选
            case 3: // 判断
                isCorrect = dbQ.getAnswer().equalsIgnoreCase(userAns.getContent());
                break;
                
            case 2: // 多选 - ⭐ Set<Character> 顺序无关比较
                isCorrect = toCharSet(dbQ.getAnswer())
                           .equals(toCharSet(userAns.getContent()));
                break;
                
            case 4: // 填空 - ⭐ trim 精确匹配
                isCorrect = dbQ.getAnswer().trim()
                           .equalsIgnoreCase(userAns.getContent().trim());
                break;
                
            case 5: // 问答 - ⭐ 跳过，score=null 待人工
                record.setIsCorrect(null);
                record.setScore(BigDecimal.ZERO);
                skipped++;
                records.add(record);
                continue;
        }
        
        // 3. 客观题写回结果
        record.setIsCorrect(isCorrect ? 1 : 0);
        BigDecimal score = isCorrect ? dbQ.getScore() : BigDecimal.ZERO;
        record.setScore(score);
        totalScore = totalScore.add(score);
        
        if (isCorrect) correct++; else wrong++;
        records.add(record);
    }
    
    // 4. 批量写 exam_answer 表
    examAnswerMapper.batchInsert(records);
    
    // 5. 更新 exam_paper
    Exam exam = examMapper.selectById(paper.getExamId());
    boolean isPass = totalScore.compareTo(new BigDecimal(exam.getScorePass())) >= 0;
    paper.setScore(totalScore);
    paper.setIsPass(isPass ? 1 : 0);
    paper.setStatus(2);
    paper.setEndTime(new Date());
    paper.setSubmitTime(new Date());
    paperMapper.updateById(paper);
    
    return new ExamResult(correct, wrong, skipped, totalScore, isPass);
}

// ⭐ 多选顺序无关比较：用 Set<Character> 避免 "ABD" vs "DBA" 误判
private Set<Character> toCharSet(String s) {
    Set<Character> set = new HashSet<>();
    for (char c : s.trim().toUpperCase().toCharArray()) {
        set.add(c);
    }
    return set;
}
```

**示例：多选 Set 比较（顺序无关）**

```text
标准答案: "ABD"  → Set{A, B, D}
学员答:   "DBA"  → Set{D, B, A} = Set{A, B, D} ✅ 正确

不用 Set 的话: "DBA" != "ABD" ❌ 误判
```

### 5.2 双保险计时机制（防作弊设计要点）

**时延分析**：

| 计时方 | 依赖 | 弱点 | 防作弊 |
|--------|------|------|--------|
| 前端倒计时 | 本地 setInterval | 改本地时间、清缓存、刷新页面 | 弱 |
| 后端计时 | 服务器时钟 | 网络延迟 | 强 |
| **双保险** | 前端先跑 + 后端兜底 | — | **最强** |

**教学点**：
- 为什么前端要倒计时？（用户体验，不能等后端每秒响应）
- 为什么后端还要再计一次？（防学员改时间）
- 如果前端倒计时完但学员没交卷？（前端自动触发 submit）
- 如果后端倒计时完但学员页面卡着？（后端定时任务自动交卷，`@Scheduled`）
