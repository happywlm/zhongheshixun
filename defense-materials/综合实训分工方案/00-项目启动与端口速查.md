## 二、端口拓扑（演示前必须核对）

| 服务 | 端口 | 用途 | 访问地址 |
|------|------|------|----------|
| **training-admin 后端** | **9898** | 管理后台接口 + **统一认证中心** | http://localhost:9898 |
| **training-api 后端** | **9899** | PC 学员 + 小程序共用业务接口 | http://localhost:9899 |
| **管理后台前端** | **5176** | Vue3 + Element Plus | http://localhost:5176 |
| **PC 学员前端** | **5174** | Vue3 学员端 | http://localhost:5174 |
| **微信小程序** | — | 微信开发者工具导入 | — |
| **MySQL** | 3306 | 数据库 | localhost:3306 |
| **Redis** | 6379 | 缓存（可选，本期未用） | — |

> ⚠️ **端口说明**：
> - 9898 是**认证中心**，管理后台 / PC 学员端的账号密码登录都走这里
> - 9899 是**业务接口**，PC 学员和小程序共用；小程序的**微信一键登录**（`/api/wx/login`）也走这里
> - 9898 与 9899 的 **JWT secret 完全相同**，所以 9898 签的 token 在 9899 也能验

## 三、启动顺序（严格按此顺序）

### 步骤 1：启动 MySQL

```bash
# Windows（如果 MySQL 是服务）
net start mysql

# 验证
mysql -u root -p -e "SHOW DATABASES;"
# 应能看到 training 数据库
```

### 步骤 2：启动 training-admin 后端（9898）

```bash
cd "D:/java spring/zhongheshixun/training-parent"

# 方式 A：Maven 启动（推荐，可调试）
mvn -pl training-admin spring-boot:run -q

# 方式 B：jar 启动（更快）
java -jar training-admin/target/training-admin-exec.jar --server.port=9898
```

**验证**：看到 `Started AdminApplication in ... seconds` 表示成功

### 步骤 3：启动 training-api 后端（9899）

```bash
cd "D:/java spring/zhongheshixun/training-parent"

# 方式 A：Maven 启动
mvn -pl training-api spring-boot:run -q -Dspring-boot.run.arguments=--server.port=9899

# 或 jar 包方式
java -jar training-api/target/training-api-exec.jar --server.port=9899
```

**验证**：看到 `Started ApiApplication in ... seconds` 表示成功

### 步骤 4：启动管理后台前端（5176）

```bash
cd "D:/java spring/zhongheshixun/training-admin/frontend"
npm run dev
```

**验证**：浏览器打开 http://localhost:5176 看到登录页

### 步骤 5：启动 PC 学员前端（5174）

```bash
cd "D:/java spring/zhongheshixun/web-student"
npm run dev
```

**验证**：浏览器打开 http://localhost:5174 看到登录页

### 步骤 6：导入微信小程序

1. 打开**微信开发者工具**
2. 点「导入项目」→ 选择 `D:\java spring\zhongheshixun\miniprogram`
3. AppID 填 **`wxf1e0f50f96da47e9`**（已注册的真实小程序号，非测试号）
4. 编译运行

> ⚠️ **真机预览必读**（手机上登录/联调必看）：
> 手机上 `localhost` 指手机自身，连不上电脑后端，会报 `request:fail net::ERR_CONNECTION_REFUSED`。`miniprogram/api/config.js` 已做动态判断——开发者工具模拟器走 `localhost`，真机走电脑局域网 IP（当前 `192.168.70.227`）。真机联调前请确认：
> - 手机与电脑连**同一个 WiFi**（不能用手机流量）
> - Windows 防火墙放行 9899 端口（管理员 PowerShell 运行）：
>   `netsh advfirewall firewall add rule name="Training API 9899" dir=in action=allow protocol=TCP localport=9899 profile=private`
> - 开发者工具「详情 → 本地设置」勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」
> - 换了 WiFi / 重启路由器导致电脑 IP 变化时，同步修改 `config.js` 里的 IP
> - 小程序登录方式为**微信一键登录**（`wx.login` 取 code → 后端 `/api/wx/login` 模拟 openid 自动建学员账号），无需输入账号密码

## 六、统一响应格式

```json
{
  "code": 200,        // 200 成功，其他失败
  "message": "success",
  "data": { ... }     // 业务数据
}
```

**常见错误码**：

| code | 含义 |
|------|------|
| 200 | 成功 |
| 401 | 未登录 / token 失效 |
| 403 | 账号已被禁用（前端判断显示）/ 无权限（Spring Security） |
| 500 | 服务器错误 |
| 1002 | 用户名或密码错误（前端 `err.code === 1002` 触发）|

## 九、常见问题排查

| 问题 | 排查步骤 |
|------|----------|
| 登录失败 | 1. 确认密码是 `123456` 不是 `password`<br>2. 确认 9898 服务已启动<br>3. 看浏览器 F12 Network 返回的 message（业务错误码 1002） |
| 页面 404 | 1. 确认前端服务已启动<br>2. 确认端口号正确（管理后台 5176，PC 学员 5174） |
| 接口 500 | 1. 看后端控制台报错<br>2. 确认数据库能连上<br>3. 确认表结构正确 |
| 接口 401 | token 过期或没带，重新登录 |
| 接口 403 | 角色权限不够，换账号 |
| 小程序白屏 / 报错 `ERR_CONNECTION_REFUSED` | 1. 确认 9899 服务已启动<br>2. **真机**：确认手机连 WiFi（非流量）、防火墙放行 9899、`config.js` 里 IP 与电脑当前局域网 IP 一致<br>3. **模拟器**：确认 `config.js` 走 localhost<br>4. 看开发者工具 / vConsole Console |
