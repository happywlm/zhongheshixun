# 学生乙 — 课程与学习（项目最重业务块）

> **综合实训角色**：学生乙  
> **业务域**：课程 + 学习  
> **核心叙事**：「课程怎么建 → 学员怎么报名 → 怎么学 → 进度怎么算 → 统计怎么看」

---

## 一、业务定位

你是整个系统的**业务主体**。你的模块是这个项目**代码量最大、接口最多、表最多**的一块，回答四个问题：

1. **课程怎么建？** — 课程 CRUD + 章节编排（药柜式组合）
2. **学员怎么报名？** — 报名课程（写 course_enroll 表）
3. **学员怎么学？** — 视频 / PDF / 文本 三种学习方式
4. **进度怎么算？** — 章节完成 → 课程进度 → 总进度

前置依赖：
- **学生甲** 提供登录 token 和 userId（报名需要用户身份）

**⭐ 学习进度计算公式（必背）**：

```
1. 章节进度 = (last_position / chapter.duration) × 100%
   上次播放位置    章节视频总时长

2. 单个章节完成 = 用户点"学完" 或 视频播放到 95% 以上
   → completed = 1

3. 课程进度 = COUNT(completed=1 的章节) / COUNT(总章节) × 100%
   已完成的章节数      该课程总章节数

4. 总学习时长 = SUM(study_duration)（所有课程累加）
```

**关键代码**：

```java
// StudyServiceImpl.updateProgress()
public void updateProgress(Long studentId, StudyProgressDTO dto) {
    // 1. 查 or 建 study_record
    StudyRecord record = studyRecordMapper.selectOne(
        new QueryWrapper<StudyRecord>()
            .eq("student_id", studentId)
            .eq("course_id", dto.getCourseId())
            .eq("chapter_id", dto.getChapterId())
    );
    if (record == null) {
        record = new StudyRecord();
        record.setStudentId(studentId);
        record.setCourseId(dto.getCourseId());
        record.setChapterId(dto.getChapterId());
    }
    
    // 2. 更新字段
    record.setLastPosition(dto.getLastPosition());
    record.setStudyDuration(
        (record.getStudyDuration() == null ? 0 : record.getStudyDuration()) 
        + dto.getDuration()
    );
    
    // 3. 计算章节进度
    CourseChapter chapter = chapterMapper.selectById(dto.getChapterId());
    double progress = (double) dto.getLastPosition() / chapter.getDuration() * 100;
    record.setProgress(Math.min(progress, 100.0));
    
    // 4. 是否完成
    if (dto.getCompleted() == 1 || progress >= 95.0) {
        record.setCompleted(1);
    }
    
    // 5. 保存
    if (record.getId() == null) {
        studyRecordMapper.insert(record);
    } else {
        studyRecordMapper.updateById(record);
    }
}
```

**关键 SQL**：

```sql
-- 查询某学员某课程进度
SELECT 
  c.id AS chapter_id,
  c.title,
  c.duration,
  COALESCE(s.progress, 0) AS progress,
  COALESCE(s.completed, 0) AS completed
FROM course_chapter c
LEFT JOIN study_record s 
  ON c.id = s.chapter_id AND s.student_id = 2
WHERE c.course_id = 1 AND c.deleted = 0
ORDER BY c.sort_order;

-- 课程总进度
SELECT 
  COUNT(CASE WHEN completed = 1 THEN 1 END) AS completed_chapters,
  COUNT(*) AS total_chapters,
  ROUND(
    COUNT(CASE WHEN completed = 1 THEN 1 END) * 100.0 / COUNT(*), 2
  ) AS progress_percent
FROM study_record 
WHERE student_id = 2 AND course_id = 1;
```

### 5.3 离线学习字段层实现（docx 3.1 强制要求）

**docx 原文**：

> 管理员在后台为每个课程建立一个属性，当该课程可以被学员下载进行离线学习时，该属性的值为一个 ZIP 文件名。由后台管理员将该课程压成 ZIP 文件上传到内容服务器该课件的目录下。在学员课程页面加入课程下载的功能，当该课件可以被下载时，学员点击下载图标即可将压缩成 ZIP 文件的课件下载到本地。

**当前实现**：字段层已实现，功能简化

```sql
-- course 表
offline_flag      TINYINT       -- 0=不可离线下载, 1=可下载
offline_filename  VARCHAR(255)  -- ZIP 文件名（NULL 表示不可下载）

-- 场景
offline_flag=0, offline_filename=NULL   → 不显示下载按钮
offline_flag=1, offline_filename='course_1.zip' → 显示下载按钮
```

**前端逻辑**：

```vue
<!-- web-student/course/detail.vue -->
<el-button 
  v-if="course.offlineFlag === 1 && course.offlineFilename"
  @click="downloadZip"
>
  ⬇ 下载离线包（ZIP）
</el-button>
```
